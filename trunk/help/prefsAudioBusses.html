Audio Prefs : Maximum Audio Busses

this is a parameter used when booting supercollider (-a). it specifies the total number of audio bus channels available. these busses are mainly internal busses not visible to the user. for example, opening a soundfile creates internal busses to route all input channels to the panorama stages.

if you open a lot of audio files simultaneously or open some sound files with a lot of channels, the server may run out of busses and displays error messages in the log window. in this case, you should increase the maximum number of audio busses.
