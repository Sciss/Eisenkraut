eisenkraut introduction

why another sound editor

eisenkraut started as demonstration example for audio programming in java. the motivation however was more deeply rooted. i found that at least on the mac platform there was no free sound editor suiting my needs.

the design goals in brief were: a simple but comfortable sound editor with support for multichannel audio and floating point encoding. i didn't want to mix the sound editing process with multitrack arrangement as is suggested by audacity, since i find it a bit pain in the ass to always "import" and "export" audio, and to browse preferences menus to change my sound file format. also i had already a great bunch of out-of-the-box classes from my meloncillo project, so the erection of version 0.1 was really quick.

in the longterm, the idea is to fuse eisenkraut and fscape (operating fscape from the process menu) and to build a robust supercollider client environment in java (the "jcollider" library). as of version 0.5, already a lot of the communication with supercollider is based on jcollider. one of the next versions shall see the possibility of building your own plug-ins in supercollider.

how to operate it

eisenkraut has been converted to a multi-document application. each soundfile represents one document, which is generated by choosing File->Open..., dragging soundfiles onto the eisenkraut icon or double clicking soundfiles in the finder, if eisenkraut is made the default soundfile application. accepted formats are aiff, wave, ircam, au/snd. note however that there is not a 100% congruency between eisenkraut and supercollider, so you might not be able to play back ircam sound files for example.

when opening a new soundfile, first of all the overview waveform is created. this is not too fast, so you may want to activate waveform caching, which will remember the waveforms of the recently used files.

editing is straight forward. on the top of the document frame, there is an editing and a transport toolbar. to edit your sound by cutting and pasting, select the pointer tool from the timeline toolbar first. press the mouse and drag to make a selection. shift+click to extend selection. ctrl+click to position insertion / playback point independant of the selection. double-click to start playback. meta+click to select all. to scroll to the selection start / end, type cursor left/right. to zoom in or out, press ctrl+left/right. to zoom vertically, type cursor up/down. see the shortcuts document for more available keyboard accelerators.

all channels of a soundfile are displayed vertically with timeline axis in horizontal direction as you expect from a classical sound editor. the display shows peak and rms energy (separately for positive and negative halfwave) in two different grey shades. when you zoom in, there will be a point when view switches to sample level display.

during editing you may often want to activate blending. (diamond shaped icon from the toolbar). blending will apply crossfades to editing operations, for example to avoid clicks when cutting.

to save your file, press meta + s. to save under a new name, choose save-as from the file menu. a rather strange dialog will pop up showing the current path name and format. to browse the harddisk, click on the small list symbol right to the path name.

sound playback

to actually hear your sound when running the transport, supercollider needs to be running. you will want to set the preferences to automatically booting supercollider when eisenkraut is launched (audio preferences). for the first time, start supercollider manually: go to the main window. press boot and wait, until the button's text changes to "terminate". if you encounter an error, check the following things:

  • open eisenkraut's preferences, go to the audio tab. check that scsynth application really points to supercollider server (the file scsynth[.exe] in your supercollider folder!).
  • note that the IP address is ignored, because scsynth is launched locally always. the port number needs usually not be changed, unless you have another supercollider application running at the same port.
  • if you get an error message saying that option -H is not understood, clear the audio device name preferences. in this case the system default audio hardware will be used.
  • note on mac, that the built in audio card doesn't support sample rates other than 44.1 kHz. set the hardware sample rate prefernces to 0 (zero) to use the valid default sample rate.

the gain slider in the server window can be used to control the master volume. this is useful for very silent sounds or for floating point files which go way above 0 dBFS, in which case you can turn down the volume, until playback is undistorted.
